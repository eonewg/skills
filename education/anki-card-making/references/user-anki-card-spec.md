# the user Anki 制卡 Skill

## 1. 技能定位

本技能用于把课程 PDF、课件、错题、复盘内容、教材片段或用户补充说明，转换为适合考研学习的 Anki 卡片。目标不是把材料机械切成问答，而是建立适合长期复习的知识网络：记住核心结论、理解机制、识别题目信号、区分易混概念，并迁移到做题。

适用范围：

- 高等数学：概念、定理、公式、题型方法、易错点、图像化理解、常见变形。
- 408 计算机基础：数据结构、计组、操作系统、计网中的概念、机制、流程、计算模板、易混辨析。
- 英文精读：长难句结构、核心表达、词组搭配、翻译逻辑、可迁移表达。
- 课程复盘：从课件或笔记中抽取知识卡、方法卡、辨析卡。
- 错题整理：不直接复制整题，优先抽取错因、题目信号、方法选择和通用步骤。

除非用户明确要求“制卡、生成 Anki、导出 APKG、整理为卡片”，否则不要主动生成 Anki 卡片；普通讲解和复盘只给正常回答。

## 2. 总原则

### 2.1 可复习性优先

Anki 卡片不是网页对话中的批量主动回忆题，而是要在长期复习中稳定、低成本地触发关键知识点。优先设计为：

- “为什么需要这个概念？”
- “看到什么题目信号时用什么方法？”
- “这个机制如何一步步运行？”
- “这个条件缺失会导致什么错误？”
- “它和相近概念有什么区别？”

不要大量制作只需照读定义的卡片；除非该内容本身只需识别性掌握。

注意：这里的“提取”是 Anki 卡片的复习效果要求，不等同于网页对话里的“主动回忆题”。网页对话可以一轮一轮问答；Anki 卡片应更像稳定的知识条目，问题清楚、答案克制、字段一致、复习成本低。

### 2.2 卡片必须自包含

每张卡片应能脱离原 PDF 或上下文独立理解。禁止使用这类问法：

- “例 3.2 中第一步为什么这样做？”
- “这页蓝框是什么意思？”
- “上面的公式如何使用？”

应改成：

- “在求形如 \(...\) 的积分时，为什么优先考虑区间再现/换元/分部积分？”
- “在操作系统中，信号量 \(S<0\) 表示什么？其绝对值有什么含义？”

### 2.3 一卡一核心点

一张卡只考一个主要提取目标。避免把多个定理、多个公式、多个易错点塞进一张卡。

可以拆分为：

- 定义卡
- 机制卡
- 条件卡
- 方法卡
- 易错辨析卡
- 反向识别卡
- 图像识别卡
- 题目信号卡

### 2.4 不直接做大题

对于例题、习题、错题，默认不把整题做成卡片，也不把完整计算过程塞进卡片。应抽离为：

- 题目信号：看到什么结构？
- 方法判断：为什么想到这个方法？
- 关键步骤：第一步/转折点是什么？
- 易错点：哪里容易误判？
- 迁移规则：以后遇到同类题怎么处理？

如果用户明确说“不要直接做题”，严格遵守，只做解题思路、题型信号、错因和通用方法。

### 2.5 内容分层处理

不是所有内容都值得制成同等强度的卡片。

- 核心机制、常考计算、易混概念：做高价值问答卡、方法卡或辨析卡。
- 常见公式、定理条件、步骤模板：做公式卡/方法卡。
- 只可能出选择题、只需识别性掌握的内容：做选择题式或填空式低负担卡。
- 过偏、过细、纯背景、低频描述：跳过或合并。
- 复杂技巧题、计算量很大的题：不做整题，抽方法。

## 3. 输入处理流程

### 3.1 先判断材料类型

拿到输入后，先判断它属于哪一类：

1. PDF/课件：需要先读取内容，必要时渲染页面核对图表、公式和蓝色框。
2. 用户文字笔记：先清理结构，再抽卡。
3. 错题/例题：先判断考点和题型，再抽通用方法卡。
4. 复盘问答：把用户答错、含糊、遗漏的点转成卡片。
5. 已有卡片：按规范优化、合并、删除或降低抽象度。

### 3.2 PDF 与课件核对规范

处理 PDF 时，不要只依赖文本抽取。遇到以下情况，必须渲染页面图像核对：

- 数学公式、分式、上下标、根号、矩阵、行列式。
- 408 中的流程图、状态转换图、时序图、存储结构图、数据通路图。
- 表格、蓝色框、总结框、图示例题。
- OCR 结果明显混乱或文本缺失。
- 用户明确要求“仔细核对 PDF”“不要漏掉蓝色框内容”。

PDF 制卡时应优先保留：

- 蓝色框/总结框中的结论。
- 老师强调的易错点。
- 图示中比文字更直观的内容。
- 题型方法与判断信号。

### 3.3 先筛选，再制卡

制卡前先做知识点筛选：

| 内容类型 | 处理方式 |
|---|---|
| 核心定义 | 做概念卡，但要问“解决什么问题/本质是什么” |
| 核心机制 | 做流程卡、原因卡、条件卡 |
| 易混概念 | 做对比辨析卡 |
| 公式/定理 | 做正向回忆、反向识别、条件限制卡 |
| 常见题型 | 做题目信号卡、方法选择卡、步骤模板卡 |
| 例题/错题 | 不整题入卡，抽方法和错因 |
| 图表/流程图 | 做图像识别卡或图文卡 |
| 选择题识别点 | 做低负担选择/填空卡 |
| 纯背景/低频描述 | 跳过或合并 |

## 4. 卡片类型规范

### 4.1 概念卡

用于核心定义、概念本质和适用场景。

推荐结构：

- 问：某概念解决什么问题？本质是什么？
- 答：一句话本质 + 关键机制 + 适用场景。
- 补充：易混点或考试提示。

示例：

- Front：操作系统中的“互斥”解决什么问题？它和“同步”的核心区别是什么？
- Back：互斥解决多个进程不能同时访问临界资源的问题；同步解决多个进程之间执行顺序或协作关系的问题。互斥强调“不能同时”，同步强调“先后配合”。

### 4.2 机制卡

用于 408 中的流程、硬件机制、操作系统机制、网络协议过程等。

推荐结构：

- 问：某机制为什么需要？如何运行？
- 答：触发条件 → 核心步骤 → 结果 → 易错点。

示例：

- Front：信号量的 P 操作为什么能实现进程阻塞？当 \(S<0\) 时表示什么？
- Back：P 操作先令 \(S=S-1\)。若结果 \(S<0\)，说明资源不足，当前进程进入阻塞队列。此时 \(|S|\) 通常表示等待该资源的进程数。

### 4.3 方法卡

用于数学题型、408 计算题和错题复盘。

固定结构：

- 题目信号：看到什么结构？
- 方法判断：为什么想到这个方法？
- 关键步骤：先做什么？
- 易错点：最容易错在哪里？

示例：

- Front：看到积分中同时出现 \(x\) 和 \(\sin x\)、\(\cos x\)，并且积分区间关于 0 对称时，优先检查什么？
- Back：优先检查奇偶性和区间对称性。若被积函数可拆成奇函数部分和偶函数部分，奇函数在对称区间积分为 0，可显著简化计算。

### 4.4 题目信号卡

用于训练“见题怎么想”。

推荐问法：

- “看到……结构，通常暗示什么方法？”
- “题目给出……条件，说明应优先检查什么？”
- “如果……，为什么不能直接用……？”

这种卡片尤其适合高数、408 计算题和错题整理。

### 4.5 易错辨析卡

用于容易混淆、容易误用的内容。

推荐结构：

- 问：A 和 B 的区别是什么？何时不能混用？
- 答：核心区别 + 判断标准 + 常见错误。

示例：

- Front：等价无穷小替换在求极限和判断级数收敛性时，为什么不能完全一样使用？
- Back：求极限时通常只关注局部比值；级数判敛依赖通项整体比较，常用极限比较法要求从某项起同号且通常取正项。对任意项级数，不能在交错结构内部随意替换，应优先考虑绝对收敛或其他判别法。

如果某张辨析卡过抽象，应改成更具体的“错误场景 + 正确做法”。

### 4.6 公式卡

公式卡不要只做“默写公式”，还要包含使用条件和反向识别。

推荐拆成：

1. 正向公式卡：给名称，回忆公式。
2. 反向识别卡：给结构，判断对应公式/方法。
3. 条件限制卡：什么时候不能用？
4. 易混公式卡：相似公式如何区分？

示例：

- Front：\(1-\cos x\) 在 \(x\to 0\) 时的常用等价无穷小是什么？
- Back：\(1-\cos x \sim \frac{x^2}{2}\)。注意它是二阶无穷小，不是 \(x\)。

### 4.7 图像识别卡

遇到图像明显优于文字时使用。适合：

- 计组数据通路图、Cache 映射图、存储芯片连接图。
- 操作系统状态转换图、同步互斥流程图。
- 计网分层、协议封装、滑动窗口图。
- 高数区域图像、积分区域、曲线关系、几何意义。

图像卡要求：

- 图片必须清晰、裁剪聚焦，不整页截图。
- Front 中提出明确问题，不只放图。
- Back 中解释图中关键结构和判断方法。
- 图片只作为辅助，不代替文字答案。

### 4.8 低负担识别卡

用于“只可能考选择题，不值得深挖”的内容。

形式可以是：

- 判断题式。
- 填空式。
- A/B 对比式。
- “看到关键词选哪个”式。

示例：

- Front：外核结构的主要优势更接近：A. 提高资源利用和灵活性；B. 简化用户程序语法；C. 增加内核态切换次数。
- Back：A。外核将资源保护和资源管理分离，让应用可更灵活地管理资源，核心考点通常是“效率/灵活性”。

## 5. 学科专项规范

### 5.1 高等数学制卡

高数卡片重点训练“为什么想到这个方法”。优先抽取：

- 函数结构信号：奇偶性、周期性、对称性、单调性、凹凸性。
- 区间信号：对称区间、可区间再现、积分上下限含参。
- 变量关系：换元、极坐标、区域投影、变量替换。
- 公式条件：泰勒展开阶数、等价无穷小适用位置、定理条件。
- 题型方法：分部积分、凑微分、三角换元、万能公式、参数法、求导法。
- 易错点：符号、区间、端点、收敛性、可导/连续/偏导关系。

复杂题不做整题，应抽成：

- “看到什么结构 → 想到什么方法”。
- “第一步为什么这么变形”。
- “这个题的关键转折是什么”。
- “以后遇到同类题如何判断”。

### 5.2 408 制卡

408 卡片应先定位模块：数据结构、计组、操作系统、计网。重点训练机制和辨析。

优先抽取：

- 机制为什么存在。
- 输入、输出和状态变化。
- 操作流程与事件顺序。
- 适用条件与边界条件。
- 相近概念区别。
- 常见选择题陷阱。
- 计算题步骤模板。

408 计算题应做成模板卡：

- Cache：Tag/Index/Offset 位数、命中流程、替换策略、写策略。
- 存储器：容量换算、地址线/数据线、芯片扩展、片选逻辑。
- 操作系统：调度算法、信号量、PV 操作、页面置换、磁盘调度。
- 计网：滑动窗口、CRC、子网划分、协议分层、时延计算。

### 5.3 英文精读制卡

英文卡片应兼顾理解和迁移，不只翻译单词。

优先抽取：

- 长难句主干。
- 从句和修饰关系。
- 逻辑连接词。
- 熟词僻义。
- 可迁移表达。
- 考研阅读中常见转折、让步、因果、评价态度。

卡片可以设计为：

- 给句子，问主干是什么。
- 给表达，问地道含义和使用场景。
- 给中文逻辑，回忆英文表达。
- 给选项，判断作者态度或逻辑关系。

## 6. 输出格式规范

### 6.1 默认交付物

默认优先交付：

1. `.apkg`：默认且唯一的制卡交付物，可直接导入 Anki。

用户只要求“看看怎么制卡”或“优化这张卡”时，不必生成 APKG；直接给改进后的卡片内容即可。

### 6.2 牌组命名

优先沿用用户指定路径。没有指定时按学科组织：

- `数学::高数::章节名`
- `408::数据结构::章节名`
- `408::计算机组成原理::章节名`
- `408::操作系统::章节名`
- `408::计算机网络::章节名`
- `英语::考研英语一::主题名`

如果用户说“继续制卡”，沿用上一轮牌组路径，不要擅自新建路径。

### 6.3 字段顺序

为适配 Anki 重复检测，默认将 `Question` 放在第 1 字段。

推荐字段：

1. `Question`
2. `Answer`
3. `Notes`
4. `Source`
5. `Tags`

可选字段：

- `Image`
- `Category`
- `Difficulty`
- `CardType`

如果使用已有模型，应保持用户当前模型字段一致；但新建模型时优先保证 `Question` 为首字段。

### 6.4 推荐模型

通用模型：

- `the user-考研通用问答卡-v1`

数学模型：

- `the user-考研数学-问答卡-v1`

408 模型：

- `the user-408-问答卡-v1`

模型不应过度复杂。优先稳定、可导入、可检索、可批量修改。

### 6.5 标签规范

每张卡至少应有：

- 学科标签：如 `数学`、`高数`、`408`、`操作系统`。
- 章节标签：如 `级数`、`进程同步`、`Cache`。
- 类型标签：如 `concept`、`method`、`mistake`、`formula`、`recognition`、`image`。

推荐附加：

- `study-action`
- `exam-signal`
- `easy-mistake`
- `core`
- `low-burden`

### 6.6 MathJax 与格式

Anki 中数学公式按 Anki 官方兼容的 MathJax 写法写入 APKG 字段：

- 行内公式：`\(...\)`
- 块级公式：`\[...\]`

这是 Anki 卡片字段里的公式格式，不按 ChatGPT 网页显示习惯或 Obsidian 习惯改写。不要使用 `$...$` 或 `$$...$$` 作为 Anki 默认公式格式，除非用户明确要求整理为 Obsidian 笔记。

避免让 LaTeX 源码裸露。尤其注意：

- `\frac{}`
- `\sqrt{}`
- `\begin{cases}`
- 矩阵、行列式、上下标。

### 6.7 HTML 使用

APKG 中允许使用 HTML 进行换行、加粗、列表和图片引用。

常用：

- `<br>` 换行。
- `<b>...</b>` 加粗重点。
- `<ul><li>...</li></ul>` 简短列表。
- `<img src="...">` 插入图片。

不要让卡片背面排版过密。长答案应拆卡。

## 7. 习题与错题转卡规范

### 7.1 不把完整题目直接做成卡

除非用户明确要求“错题卡要保留题干”，否则不要把完整大题放入 Anki。原因：

- 复习成本高。
- 容易变成重新做题，而不是提取方法。
- 一张卡承载信息过多，复习时难以判断掌握情况。

### 7.2 例题应抽象成方法

例题处理模板：

1. 这题考什么？
2. 题干中哪个结构提示方法？
3. 第一步为什么这样做？
4. 中间的关键转折是什么？
5. 最容易错在哪里？
6. 同类题如何迁移？

最终制成 1–4 张方法卡，而不是一整张题解卡。

### 7.3 错题应抽错因

错题卡优先记录：

- 错因类型：概念不清、方法没想到、条件漏看、符号错误、计算错误、题型识别失败。
- 正确触发信号。
- 下次遇到的检查动作。

示例：

- Front：做积分题时，如果第一眼没思路，应优先从哪些信号排查？
- Back：先看区间对称性、奇偶性、周期性、被积函数结构、是否适合换元、是否有可凑微分项、是否能分部积分、是否可转化为已知基本积分。

## 8. 卡片数量控制

### 8.1 不追求“全覆盖式爆量”

制卡不是把 PDF 每句话都做成卡。应控制数量，优先高价值卡。

一般建议：

- 一小节：8–20 张。
- 一讲课：20–60 张。
- 机制密集章节：可更多，但必须分组。
- 只需选择题识别的内容：少量低负担卡即可。

### 8.2 避免一次复习压力过大

如果用户是在网页对话中进行复盘，一轮主动回忆题控制在 5–10 个。这是对话复盘规则，不是 Anki 制卡字段规则；制卡可以批量生成，但每张卡仍要短小、清晰、可复习。

## 9. 卡片质量检查清单

生成前逐张检查：

1. 是否自包含？
2. 是否一张卡只考一个核心点？
3. 是否能在复习时触发明确的知识点，而不是只看答案？
4. 是否避免直接复制大题？
5. 是否包含必要条件和适用范围？
6. 是否指出容易混淆或容易误用的点？
7. 数学公式是否使用 Anki MathJax？
8. `Question` 是否在第 1 字段？
9. 标签是否足够检索？
10. 图片是否清晰、裁剪聚焦？
11. 卡片是否过长，是否需要拆分？
12. 是否保留了 PDF 中蓝框、图表、老师强调内容？

## 10. 常见坏卡与修改方式

### 10.1 太抽象

坏卡：

- Front：等价无穷小替换用于级数时要注意什么？
- Back：要注意正性和绝对值。

改法：加入错误场景和操作规则。

- Front：为什么不能像求极限一样，在交错级数的通项内部随意做等价无穷小替换？
- Back：级数判敛比较的是通项形成的整体求和行为。若用极限比较法，通常要保证从某项起同号且可按正项比较；交错级数还涉及符号抵消，随意替换可能破坏单调性、符号结构或判别条件。应先判断绝对收敛，或使用交错级数、Dirichlet、Abel 等判别法。

### 10.2 太像直接做题

坏卡：

- Front：计算第 3 题的积分。
- Back：完整题解步骤。

改法：抽题型方法。

- Front：当积分中出现 \(x f(\cos x)\)，且区间为 \([0,\pi]\) 时，为什么常考虑区间再现？
- Back：因为换元 \(x\mapsto \pi-x\) 后，\(\cos(\pi-x)=-\cos x\)，可与原积分相加或相减，消去难处理部分或构造对称关系。

### 10.3 答案过长

坏卡：一张卡背面放完整章节总结。

改法：拆成概念卡、机制卡、条件卡、易错卡、方法卡。

### 10.4 只会识别不需要深挖

坏卡：对低频选择题内容做很长问答。

改法：做低负担识别卡，例如判断、填空、选择题式卡。

## 11. 执行输出模板

当用户要求制卡时，先按以下顺序执行：

1. 明确材料范围和牌组路径。
2. 读取/核对材料，必要时渲染 PDF 页面。
3. 提取知识点并筛选制卡价值。
4. 按卡片类型拆分。
5. 生成 APKG。
6. 给出简短说明：卡片数量、牌组名、覆盖内容、导入方式、排除内容。

说明模板：

- 牌组：`...`
- 卡片数：`...`
- 类型：概念卡 / 方法卡 / 易错辨析卡 / 公式卡 / 图像卡 / 识别卡
- 已排除：过长整题、低价值背景、重复定义
- 附件：APKG

## 12. 与用户互动时的规则

- 用户说“这张太抽象”：改成具体场景 + 错误示例 + 正确规则。
- 用户说“这张可以删吗”：判断它是否可通过做题自然掌握；若是低价值抽象卡，建议删或降级为低负担卡。
- 用户说“不要直接做题”：只抽方法、信号、错因，不输出完整题解卡。
- 用户说“继续制卡”：沿用上一轮牌组路径和字段规范。
- 用户要求“适合 Obsidian”：才改用 Obsidian 公式格式 `$...$` 和 `$$...$$`。
- 用户要求“批量主动回忆”：这是网页对话复盘流程，一轮 5–10 个问题，不要一次给过多；不要把这种互动题直接混同为 Anki 卡片。
- 用户要求“核对 PDF”：必须结合页面渲染检查公式、图、表和蓝框。

## 13. 最终目标

每张卡都应服务于以下至少一个目标：

1. 记住核心结论。
2. 理解机制为什么存在。
3. 看到题目能识别方法信号。
4. 避免常见误用和混淆。
5. 能把一个例题迁移到同类题。
6. 降低复习负担，而不是制造新的信息噪音。

制卡时宁可少而准，不要多而散。核心标准是：复习这张卡后，用户是否更容易在考试题、课后题或复盘中独立想起正确方法。
