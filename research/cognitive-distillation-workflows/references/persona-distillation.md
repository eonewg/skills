# Persona / Thinker Distillation

Use when the target is a person, thinker, creator, founder, educator, public figure, or theme-persona whose value is mostly in how they think.

## Research tracks
Write each track to `references/research/`.

1. `01-writings.md` — books, long essays, papers, newsletters, repeated claims, self-coined terms, recommended canon.
2. `02-conversations.md` — interviews, podcasts, talks, AMAs, spontaneous reasoning, changes of mind, refusal patterns.
3. `03-expression-dna.md` — short-form posts, vocabulary, sentence rhythm, humor, certainty level, metaphors.
4. `04-external-views.md` — biographies, criticism, peer accounts, disagreement, public perception.
5. `05-decisions.md` — major choices, turning points, product/company/investment decisions, action vs stated principle.
6. `06-timeline.md` — life/work timeline, phase changes, recent changes.

Quick tier may use tracks 1, 2, 3 only. Standard uses all six. Deep adds local material and long-form transcript extraction.

## Synthesis method
Extract candidate principles, then filter them:

- Cross-context recurrence: appears in multiple domains or episodes.
- Generative power: helps predict how the person would reason on a new problem.
- Distinctiveness: not a generic smart-person platitude.

Classify output:

- 3-7 mental models: deep lenses with evidence and limits.
- 5-10 decision heuristics: if-situation-then-judgment rules.
- Expression DNA: tone, rhythm, metaphors, certainty, taboo phrases.
- Values and anti-patterns: what they optimize for and what they reject.
- Knowledge lineage: who influenced them, who they influenced, adjacent schools.
- Honest boundaries: what the distillation cannot know or should not claim.

## Answer protocol for generated persona skills
Generated skills should not only “sound like” the person. They need a protocol:

1. Classify the user question: factual, framework-only, or mixed.
2. For factual/mixed questions, gather current facts before applying persona models.
3. Apply the person’s models and heuristics.
4. State uncertainty when the source base is thin.
5. Separate “known from sources” from “inferred from the model.”

## Quality gates
Before delivery, test with:

- one known historical question where the person’s real stance is documented;
- one edge question outside their domain, where the skill should show restraint;
- one practical user task, where the skill should produce useful guidance rather than cosplay.
