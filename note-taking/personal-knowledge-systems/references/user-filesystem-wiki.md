# the user filesystem wiki ingestion

Use this when saving web clips, article extractions, transcripts, or other source material into the user's local wiki.

## Known wiki path

- Wiki root: `~/wiki`
- Schema: `~/wiki/SCHEMA.md`
- Index: `~/wiki/index.md`
- Log: `~/wiki/log.md`
- Legacy path `~/.hermes/data/wiki` was merged into the canonical wiki on 2026-05-29 and should be a symlink to `~/wiki`. Do **not** treat `.hermes/data/wiki` as a separate wiki; if it ever stops being a symlink, merge/redirect it before answering wiki-backed questions.

## Query-before-advice rule

When the user asks for planning/timeline/recommendation in a domain that has wiki coverage, the wiki is the first source of truth, not generic reasoning. This matters especially for kaoyan/408/math planning and AI/Agent concepts that were previously ingested from articles or Zhihu experience posts. Search/read the relevant formal page first (for 408, start at `concepts/cs408-prep.md`; for math, `concepts/math-prep.md` / `concepts/zhangyu-math-learning-track.md`; for Agent/AI workflow, use `index.md` plus the nearest concept page), then give the answer and cite the wiki-derived basis in plain language. If you instead infer from memory and the user asks “why didn’t you check wiki?”, treat it as a workflow miss and correct by reading the wiki immediately.

## Required pattern

0. **Duplicate and low-signal triage before ingestion:**
   - Before processing, check if the source URL already appears in any raw article's frontmatter. If it exists, the article is already ingested -- do not re-ingest; just tell the user and optionally share your take.
   - After fetching the content, assess signal density before deciding on concept-page depth:
     - **High signal** (methodologies, frameworks, industry analysis, durable principles) -- full workflow: concept page + patch existing + index/hot/manifest/lint.
     - **Low signal** (curated book lists, mood pieces, lightweight roundups, announcements, photo galleries, hobby/sports gear news, celebrity/player equipment-change notes) -- minimal ingest only: save raw, update hot.md + manifest, run lint. Do not create a concept page or patch existing concept pages. If it has one reusable phrase or connection, that belongs in the hot.md entry, not a formal page. For hobby/sports gear articles, preserve the practical takeaway (e.g. “do not blindly copy pro equipment; stabilize your own setup first”) in hot.md rather than creating a formal equipment page.

1. Read `SCHEMA.md` before writing if the format is not already in context.
2. the user's default preference for shared 公众号/articles/PDFs/videos: **first send him a concise重点提炼 in chat**, then compile the useful information into the wiki. Do **not** skip the user-facing digest and jump straight to archive work. After the digest, do **not** only save the raw source, and do **not** stop at a chat summary. The expected full output is: chat digest → raw capture → one or more durable formal pages that make the content reusable later.
   - If the user sends only a URL with no instruction, infer the standard flow as "提炼重点 + 入库", and do it in one go: send the digest, then immediately proceed to the full ingestion (raw + concept + patches + manifest + lint). Do **not** pause after the digest to ask "要搞吗？/开干？" — the user will stop you if he doesn't want it ingested. The digest is the user-facing value; the archive work is implied.
   - If an archive/write step is blocked by approval or interrupted, report exactly what was completed (e.g. extracted/read/digested) and what remains pending; do not imply the raw/concept/index/manifest updates happened until they have been verified.

Search-engine snippets, `r.jina.ai`, and plain headless browser access are fallback discovery/triage only and should not be treated as the final archival evidence when Kimi is available. Do not rely only on chat summary; create a focused concept page when the article contains durable study strategy, planning, or reusable methods. For high-signal 考研数学/408/公共课经验贴, typical destination is a planning/method page under `concepts/`, plus patches to `concepts/math-prep.md`, `concepts/cs408-prep.md`, `concepts/english-prep.md`, `concepts/politics-prep.md`, and `concepts/kaoyan-overview.md` so later questions can route from the user's active study state.

5. Store original/source material under `raw/` without rewriting the body semantics.
   - Web articles normally go under `raw/articles/`.
   - Transcripts normally go under `raw/transcripts/`.
6. Add YAML frontmatter for raw captures:
   - `source_url`
   - `ingested`
   - `sha256` of the raw body, excluding the extraction tool's frontmatter when present
   - add useful metadata such as `source_title`, `source_author`, `published` when available.
   - Compute and verify the hash from the final raw file exactly as stored after removing only the wiki YAML frontmatter. Match `scripts/wiki_lint.py` precisely: parse with `^---\n(.*?)\n---\n(.*)$` and hash group 2, so the newline immediately after the closing frontmatter is included if present. Avoid pre-write variants such as extra `.lstrip()` or different leading-newline handling, which can produce confusing hash mismatches.
   - When a one-off ingestion script writes raw frontmatter, prefer a two-pass write: first write the raw file with a temporary `sha256: 0`, read that final file back, recompute the hash exactly from the parsed body, then patch both raw frontmatter and `.manifest.json`. Computing from the extractor body before adding final wiki frontmatter can still drift by one leading newline and create a new raw hash warning.
   - For local PDFs/documents, keep the original file under `raw/documents/` with a lowercase kebab-case English filename, then create a sibling extracted raw Markdown file whose frontmatter includes `source_type: pdf` and `original_pdf: raw/documents/<file>.pdf`. The formal page should cite the Markdown raw capture, not the binary PDF directly.
   - For local Markdown/text study handouts, also capture under `raw/documents/` with `source_type: markdown` or `source_type: text`, preserve the original body in a fenced block, and add a normalized excerpt only for queryability. The formal page should cite the raw Markdown capture and use a lower-case English kebab-case slug.
   - For compact kaoyan math formula sheets, do not merely archive formulas. Create a reusable `concepts/` tool page with: `什么时候用这页`, `最小动作`, grouped formula tables, usage rules, high-frequency pitfalls, practice conversion, and wikilinks back to `[[math-prep]]`, `[[learning-system]]`, and adjacent formula pages. Also update `concepts/math-prep.md`, `index.md`, `hot.md`, `.manifest.json`, and `log.md`. When the new formula page strengthens an existing adjacent page (e.g. trigonometric identities strengthening basic integrals / equivalent infinitesimals / Taylor expansions), update that adjacent page's `sources` and reciprocal wikilinks instead of leaving the new page isolated.
7. Create or update a formal page under the right class folder:

When appending a recent-source note, check the body separately from the YAML `sources:` list; a source path appearing in frontmatter does **not** mean the human-readable hot entry was added.
   - `.manifest.json`: track ingested raw sources by path/url, sha256, updated formal pages, ingestion time, and status (`new`, `modified`, `unchanged`, `deleted`) to prevent duplicate ingestion and missed updates.

Verification should include: index link appears exactly once, new weekly heading appears exactly once, log entry appears, source files no longer remain in `.hermes/out`, moved files are visible in Trash, derived layers (`facts/`, `graph/semantic_embeddings*`, reports) were rebuilt after the write, and `scripts/wiki_lint.py` has `issue_count: 0` (legacy raw-hash warnings are non-blocking for this workflow). Cron/user-facing reports for this workflow should use short lists, not Markdown tables.

## Lint pitfalls

- **bad_tags**: Tags must match SCHEMA.md's Tag Taxonomy exactly. Tags like `attention`, `architecture` not in the taxonomy list will be flagged. Only use tags already listed; if a new tag is needed, add it to SCHEMA.md's `## Tag Taxonomy` first.
- **hash mismatch**: When computing SHA256 for raw captures, match `scripts/wiki_lint.py` precisely: parse with `^---\\n(.*?)\\n---\\n(.*)$` regex and hash group 2 (the body after frontmatter, including the leading newline). Avoid pre-write variants like `.lstrip()` or different newline handling.
