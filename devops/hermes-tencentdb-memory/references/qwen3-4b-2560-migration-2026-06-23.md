# 2026-06-23: Qwen3-Embedding-4B / 2560D migration notes

## What changed

the user chose to reduce daily embedding cost by migrating both TencentDB Memory and LLM Wiki from SiliconFlow `Qwen/Qwen3-Embedding-8B` / 4096D to `Qwen/Qwen3-Embedding-4B` / 2560D.

Why 2560D instead of 2048D:
- Main savings come from 8B → 4B pricing, not from shaving dimensions.
- 2560D is the 4B model's maximum dimension and keeps more quality margin.
- It is still much smaller than 4096D and keeps the two knowledge systems aligned.

## Verified procedure

1. Back up live files before touching config:
   - `~/.memory-tencentdb/memory-tdai/tdai-gateway.json`
   - `~/.memory-tencentdb/memory-tdai/vectors.db`
   - `~/wiki/graph/semantic_embeddings.{json,npz}`
   - relevant wiki scripts if editing defaults.

2. Probe the endpoint without printing secrets:
   - POST `https://api.siliconflow.cn/v1/embeddings`
   - body: model `Qwen/Qwen3-Embedding-4B`, dimensions `2560`, `encoding_format: float`
   - expected: all returned embeddings length `2560`.

3. Configure TencentDB through the Hermes-mode ctl script:

```bash
SCRIPT=~/.openclaw/npm/node_modules/@tencentdb-agent-memory/memory-tencentdb/scripts/memory-tencentdb-ctl.sh
bash "$SCRIPT" --hermes config embedding \
  --provider openai \
  --api-key "$SILICONFLOW_API_KEY" \
  --base-url "https://api.siliconflow.cn/v1" \
  --model "Qwen/Qwen3-Embedding-4B" \
  --dimensions 2560 \
  --restart
```

4. Verify immediately whether vectors were repopulated. In this session, the gateway recreated `l1_vec` / `l0_vec` at `float[2560]` and updated `embedding_meta`, but row counts were zero:
   - `l1_records=180`, `l1_vec_rowids=0`
   - `l0_conversations=2226`, `l0_vec_rowids=0`

5. If vec row counts are zero or lower than metadata counts, stop the sidecar and run the reusable support script:

```bash
SCRIPT=~/.openclaw/npm/node_modules/@tencentdb-agent-memory/memory-tencentdb/scripts/memory-tencentdb-ctl.sh
bash "$SCRIPT" --hermes stop
node ~/.hermes/skills/devops/hermes-tencentdb-memory/scripts/reindex-siliconflow-sqlite-vec.mjs \
  --model Qwen/Qwen3-Embedding-4B \
  --dimensions 2560
bash "$SCRIPT" --hermes start
```

The script uses Node's `node:sqlite` with `{ allowExtension: true }` and loads `sqlite-vec` from the TencentDB plugin install. This avoids the `enableLoadExtension(true)` invalid-state pitfall.

6. After reindex, verify:
   - `embedding_meta`: model `Qwen/Qwen3-Embedding-4B`, dimensions `2560`
   - `l1_vec`: `float[2560]`
   - `l0_vec`: `float[2560]`
   - `l1_vec_rowids == l1_records`
   - `l0_vec_rowids == l0_conversations`
   - `/health`: `vectorStore=true`, `embeddingService=true`
   - recall logs: `[hybrid-embedding] Embedding OK, dims=2560`

7. Rebuild LLM Wiki semantic embeddings after TencentDB config is updated. Its semantic build script reads `tdai-gateway.json`, so once defaults are patched, `python3 scripts/wiki_v2_build.py` should generate `graph/semantic_embeddings.npz` with shape `[doc_count, 2560]`.

8. Run validation:
   - `python3 scripts/wiki_lint.py`
   - `python3 scripts/wiki_v2_weekly_health.py`
   - `python3 scripts/wiki_v2_eval.py`

9. Optional cleanup only after all counts and health checks pass:
   - remove migration backups if user asks
   - remove temporary one-off reindex scripts from `~/.memory-tencentdb/`
   - `VACUUM` `vectors.db` after vector dimensionality reduction to reclaim free pages.

## Pitfalls learned

- Do not trust config drift handling alone. It may rebuild the schema but leave vector tables empty.
- Always compare vec row counts against metadata row counts before declaring success.
- After reducing dimensions, file size may not shrink until `VACUUM` runs.
- If a multi-file patch fails validation, subsequent `skill_manage` patches can still succeed; verify the final disk content with read-back/counts rather than relying on the failed patch event.
